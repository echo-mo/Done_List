# Context / 背景
edit按钮是一个用来修改任务内容的功能模块。请阅读项目中与“编辑任务”相关的代码文件。

# Bug Description / 缺陷描述
我在测试时发现一个主要在**手机端**出现的交互 bug：
1. 触发操作：点击“Edit”按键进入编辑状态。
2. 异常表现：手机软键盘弹起，开始输入内容时，编辑状态会自动退出（或弹窗/输入框直接关闭）。
3. 结果：导致无法正常录入修改内容。

# Expected Behavior / 期望行为
在手机端点击编辑并输入时，组件应该稳定保持在编辑状态。软键盘的弹起不应导致状态丢失，直到用户明确触发“保存”或“取消”操作。

# Potential Causes to Check / 建议排查方向
请结合我的代码，重点排查以下几种可能：
1. 视口变化与重渲染：手机软键盘弹起导致窗口高度改变，是否触发了父级页面的异常重渲染，从而重置了当前组件的“编辑中”状态？
2. 焦点丢失误触：输入框的 `onBlur` 或外部点击事件（Click Outside）是否逻辑过于敏感，导致在输入时被意外触发，执行了退出编辑的逻辑？
3. 表单默认行为：如果使用了 `<form>` 或 `<button>`，是否存在没有被 `preventDefault()` 拦截的默认提交/刷新行为？
